## 拆分规则（必须严格遵守）

1. **routers 层**：只负责解析 HTTP 请求参数 → 调用对应 service → 返回响应。禁止在 router 里直接操作数据库 session 或写业务逻辑。

2. **services 层**：接收具体参数（不接收 Request 对象），操作数据库，返回 ORM 对象或 Python 基本类型。service 函数不能 import 任何 fastapi 模块。

3. **models 层**：只放 SQLAlchemy ORM class 定义，不放 Pydantic schema。

4. **schemas 层**：只放 Pydantic BaseModel，不 import SQLAlchemy。

5. **config.py**：所有常量（DATABASE_URL、API_KEY、BASE_URL）改为从环境变量读取，同时提供默认值便于开发。同时生成对应的 `.env.example` 文件。

6. **database.py**：删除重复的 `get_session`，统一使用 `get_db`，放在此文件中。

7. **chat router 的特殊处理**：`event_generator` 中"流式输出结束后保存消息"的逻辑，将"创建 conversation"和"保存 message"抽取到 `conversation_service` 的独立函数中，`llm_service.stream_chat` 只负责 LLM 通信，chat router 负责编排两者。
