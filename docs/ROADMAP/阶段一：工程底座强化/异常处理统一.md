# 异常处理统一

覆盖 LLM 超时、流式响应中途失败、JSON 解析失败三类场景，统一 SSE 事件格式，并对 messages 表增加状态字段。

---

## 1. SSE 统一事件 Schema

所有 SSE 事件切换为 type-based 结构，不保留旧格式兼容。

```
data: {"type": "delta", "content": "..."}   # 增量内容
data: {"type": "done"}                       # 正常结束
data: {"type": "error", "message": "..."}    # 异常结束
```

前端同步改 parser：
- `type = delta` → 追加内容
- `type = done` → 标记完成，停止 loading
- `type = error` → 标记失败，显示错误提示和重试按钮

`schemas/dto.py` 中定义 `SseDelta`、`SseDone`、`SseError` 三个 Pydantic model，以及 `sse()` 辅助函数统一序列化为 `data: ...\n\n` 格式。

这套格式在阶段二可以直接扩展 RAG 事件流，如 `type = retrieval_start`、`type = citation` 等，无需改协议。

---

## 2. LLM 超时与重试

```
超时配置（httpx.Timeout）：connect=5s，read=60s，write=10s，pool=5s
重试次数：最多 2 次，间隔 1s（asyncio.sleep）
重试条件：httpx.TimeoutException / httpx.NetworkError / HTTP 5xx
不重试：HTTP 4xx（参数错误，重试无意义）
```

超时重试封装在 `services/llm.py`，`stream_chat` 内部处理，对外只暴露 `LLMError`。

所有重试耗尽后仍失败，raise `LLMError`，由 `routers/ai_chat.py` 的 `event_generator` 捕获，发送 `type=error` 事件后结束流。

---

## 3. 流式响应中途失败处理

SSE 连接一旦建立，HTTP 状态码已是 200，无法再改。中途失败通过 `type=error` 事件通知前端。

```
LLM 完全未输出 → 发 type=error → assistant_content 为空，不落库 assistant message
LLM 输出一半后失败 → 发 type=error → 落库 partial content，status=failed
LLM 正常完成 → 发 type=done → 落库，status=completed
```

不持久化 `streaming` 状态，流结束前不写库，避免 crash 留下僵尸记录。

---

## 4. messages 表新增字段

```
status:        VARCHAR(20)  NOT NULL DEFAULT 'completed'   -- completed / failed
error_message: TEXT         NULL
```

通过 Alembic 0002_messages_status 迁移添加。

落库逻辑在 `services/message_saver.py`，`routers/ai_chat.py` 的 background task 传入 `status` 和 `error_message`。

---

## 5. 全局异常 Handler

在 `main.py` 注册 FastAPI exception handler，捕获未处理的异常统一返回 JSON：

```json
{"detail": "Internal server error", "request_id": "uuid"}
```

SSE 路由内部自己处理，不依赖全局 handler（全局 handler 无法在 streaming 中途介入）。

---

## 实现文件

| 文件 | 变更 |
|------|------|
| `schemas/dto.py` | 新增 SseDelta / SseDone / SseError Pydantic schema 和 sse() 辅助函数 |
| `services/llm.py` | 超时配置（httpx.Timeout）、最多 2 次重试、统一抛出 LLMError |
| `routers/ai_chat.py` | event_generator 捕获 LLMError，发 type=error 事件；传 status/error_message 给 background task |
| `services/message_saver.py` | 接收 status / error_message 参数，partial 内容以 failed 状态落库 |
| `models/orm.py` | Message 模型新增 status、error_message 字段 |
| `main.py` | 注册全局 exception handler，返回带 request_id 的 JSON |

---

## 验证结果

- 正常流：事件序列为 `delta... → done`，status=completed 落库
- LLM 失败流：事件序列为 `delta(partial)... → error`，partial content 以 status=failed 落库，error_message 记录原始错误
- SSE schema 格式：`data: {"type":"delta","content":"..."}` / `data: {"type":"done"}` / `data: {"type":"error","message":"..."}`
