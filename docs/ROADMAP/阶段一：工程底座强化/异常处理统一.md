# 异常处理统一

覆盖 LLM 超时、流式响应中途失败、JSON 解析失败三类场景，统一 SSE 事件格式，并对 messages 表增加状态字段。

---

## 1. SSE 统一事件 Schema

所有 SSE 事件切换为 type-based 结构，不保留旧格式兼容。

```
data: {"type": "delta", "content": "..."}   # 增量内容
data: {"type": "done"}                       # 正常结束
data: {"type": "error", "message": "..."}    # 异常结束
```

前端同步改 parser：
- `type = delta` → 追加内容
- `type = done` → 标记完成，停止 loading
- `type = error` → 标记失败，显示错误提示和重试按钮

这套格式在阶段二可以直接扩展 RAG 事件流，如 `type = retrieval_start`、`type = citation` 等，无需改协议。

---

## 2. LLM 超时与重试

```
超时配置（httpx）：connect=5s，read=60s，write=10s
重试次数：最多 2 次，retries 间隔 1s
重试条件：网络错误 / httpx.TimeoutException / HTTP 5xx
不重试：HTTP 4xx（参数错误，重试无意义）
```

超时重试封装在 `services/llm.py`，`stream_chat` 内部处理，对外不感知重试过程。

所有重试耗尽后仍失败，向 generator 抛出异常，由 `routers/ai_chat.py` 的 `event_generator` 捕获，发送 `type=error` 事件后结束流。

---

## 3. 流式响应中途失败处理

SSE 连接一旦建立，HTTP 状态码已是 200，无法再改。中途失败通过 `type=error` 事件通知前端。

```
LLM 完全未输出 → 发 type=error → 不落库 assistant message
LLM 输出一半后失败 → 发 type=error → 落库 partial content，status=failed
LLM 正常完成 → 发 type=done → 落库，status=completed
```

不持久化 `streaming` 状态，流结束前不写库，避免 crash 留下僵尸记录。

---

## 4. messages 表新增字段

```
status:        VARCHAR(20)  NOT NULL DEFAULT 'completed'   -- completed / failed
error_message: TEXT         NULL
```

用 Alembic 迁移添加（不手动 ALTER TABLE）。

落库逻辑在 `services/message_saver.py`，调用方（`routers/ai_chat.py` 的 background task）传入 `status` 和 `error_message`。

---

## 5. 全局异常 Handler

在 `main.py` 注册 FastAPI exception handler，捕获未处理的异常统一返回 JSON：

```
{"detail": "Internal server error", "request_id": "..."}
```

对于 SSE 路由以外的普通接口，走标准 HTTP 错误响应。
SSE 路由内部自己处理，不依赖全局 handler（全局 handler 无法在 streaming 中途介入）。

---

## 实现文件

| 文件 | 变更 |
|------|------|
| `services/llm.py` | 添加超时配置、重试逻辑，抛出统一异常类型 |
| `routers/ai_chat.py` | event_generator 捕获异常，发 type=error 事件；调用 message_saver 时传 status |
| `services/message_saver.py` | 接收 status / error_message 参数，写入 messages 表 |
| `models/orm.py` | Message 模型新增 status、error_message 字段 |
| `main.py` | 注册全局 exception handler |
| `schemas/dto.py` | 新增 SSE 事件 Pydantic schema（SseDelta / SseDone / SseError） |
| Alembic migration | 添加 messages.status 和 messages.error_message 字段 |
