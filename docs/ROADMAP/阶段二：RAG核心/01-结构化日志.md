# 结构化日志

> **状态：✅ 已完成**

在 RAG ingestion pipeline 开始前补上，方便排查哪一步卡住、哪个 chunk 失败。

---

## 方案

标准库 `logging` + 自定义 `JsonFormatter`，不引入 structlog（零依赖，够用）。

每条日志输出为单行 JSON，字段：

```json
{
  "timestamp": "2026-05-13T10:00:00Z",
  "level": "INFO",
  "logger": "services.ingestion",
  "request_id": "uuid",
  "user_id": "uuid",
  "event": "chunk_embedded",
  "doc_id": "uuid",
  "chunk_index": 3,
  "duration_ms": 42
}
```

---

## request_id 传递

FastAPI middleware 为每个请求生成 `request_id`，存入 `contextvars.ContextVar`，日志里直接读取，无需手动传参。

Background task（ingestion pipeline）继承调用方的 ContextVar 上下文，request_id 自动传递。

---

## 日志级别约定

| 级别 | 场景 |
|------|------|
| INFO | 正常完成：chunk 入库、检索命中、请求耗时 |
| WARNING | 重试发生、BM25 索引重建、chunk 解析异常跳过 |
| ERROR | ingestion 失败、embedding API 报错、向量检索异常 |
| DEBUG | 开发阶段详细字段，生产不输出 |

---

## 实现文件

| 文件 | 说明 |
|------|------|
| `core/logging.py` | JsonFormatter + `setup_logging()` 初始化函数 |
| `core/middleware.py` | request_id 生成中间件，写入 ContextVar |
| `main.py` | 启动时调用 `setup_logging()`，注册中间件 |
