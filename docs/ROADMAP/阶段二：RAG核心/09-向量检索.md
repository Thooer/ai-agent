# 向量检索

> **状态：✅ 已完成**

pgvector cosine 相似度检索，按 user_id 隔离，top-k 可配置。

---

## SQL

```sql
SELECT
    c.id,
    c.chunk_index,
    c.content,
    c.start_char,
    c.end_char,
    c.doc_id,
    d.name AS doc_name,
    1 - (c.embedding <=> $1::vector) AS score
FROM chunks c
JOIN documents d ON c.doc_id = d.id
WHERE c.user_id = $2
  AND c.embedding IS NOT NULL
ORDER BY c.embedding <=> $1::vector
LIMIT $3;
```

`<=>` 是 pgvector 的 cosine 距离算子（越小越相似）。`1 - distance` 转为相似度分。

## 接口

```python
async def vector_search(
    user_id: UUID,
    query_embedding: list[float],
    top_k: int = 10,
) -> list[VectorResult]:
    ...

@dataclass
class VectorResult:
    chunk_id: UUID
    doc_id: UUID
    doc_name: str
    chunk_index: int
    content: str
    start_char: int
    end_char: int
    score: float   # cosine 相似度，越大越好
    rank: int
```

## IVFFlat 索引说明

- `lists=100`：适合 chunk 数 < 100 万
- 查询时 pgvector 自动使用索引，无需额外配置
- 召回率不是 100%（近似检索），比精确检索快得多；面试能讲与 HNSW 的取舍
