# 文档解析

上传文件 → 提取纯文本，支持 PDF / Markdown / TXT 三种格式。

---

## 格式支持

| 格式 | 库 | 说明 |
|------|----|------|
| TXT | 内置 | 直接读，处理编码（UTF-8 优先，fallback GBK） |
| Markdown | 内置 | 直接读原始文本，不渲染 HTML（chunk 保留 markdown 语法，对 embedding 影响可忽略） |
| PDF | `pymupdf`（fitz） | 按页提取文本，过滤空页，不处理图片/表格 |

不做 Word / HTML / 图片 OCR，面试能讲"有意识排除低信噪比内容"。

---

## 上传接口

```
POST /documents/upload
Content-Type: multipart/form-data
Authorization: Bearer <token>

file: <binary>
```

响应：
```json
{
  "doc_id": "uuid",
  "task_id": "uuid",
  "message": "ingestion started"
}
```

接口职责：保存文件元信息到 documents 表，创建 Redis task，启动 background task 跑 ingestion pipeline，立即返回 task_id。前端用 task_id 轮询进度。

---

## 文件大小限制

FastAPI 默认无限制，在接口层加 `UploadFile` 校验：
- 最大 20MB（当前阶段文档场景够用）
- 不支持的格式返回 422

---

## 解析结果

返回 `ParsedDocument`：

```python
@dataclass
class ParsedDocument:
    text: str          # 完整纯文本
    char_count: int
    page_count: int    # PDF 才有，其他为 1
```

解析失败（加密 PDF、损坏文件）raise `ParseError`，ingestion pipeline 捕获后标记 task failed。
